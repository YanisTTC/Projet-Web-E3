Stud'Path - Report

Authors: Luisa Anjos & Yanis Hamani

Computer Science, Design, Architecture, and Development Program

1.1 Context and Motivation

Finding adequate housing close to educational institutions is a common challenge for students in France. This project addresses this issue by providing an interactive map that visualises the locations of student housing, universities and any other desired address, making it easier to analyse proximity, accessibility, and distribution patterns.

This tool is not only designed for students but also for policymakers and researchers. Policymakers can use it to identify underserved areas and plan infrastructure improvements, while researchers can gain insights into urban planning and education accessibility trends. By translating raw geospatial data into an interactive and user-friendly platform, this project transforms complex datasets into actionable information.

1.2 Objectives

The primary goal of this project is to design and develop an interactive web-based map that allows users to explore the geographic distribution of student housing and higher education institutions in France, and compare different routes. This map offers the following functionalities:

  • Visualising the locations of student housing and universities using geospatial layers.
  • Enabling users to filter and analyse data based on different criteria.
  • Providing an intuitive interface with features like zooming, panning, and pop-ups containing detailed information about each point of interest.
  • Incorporating multi-language support for accessibility to a broader audience.
  • Offering route comparison tools to help students evaluate travel times between housing and universities.

2.1 Overview of Data

This project utilises two primary datasets in the GeoJSON format:

  1. Student Housing Locations: This dataset contains geographic information about student housing across France, including precise coordinates and descriptive details.
  1. Higher Education Institutions: This dataset provides the locations and attributes of universities and other higher education institutions in France.

Both datasets were sourced from opendatasoft. The use of GeoJSON ensures compatibility with web-based geospatial visualization tools like Leaflet.js. URLs for these datasets are:

  • Student Housing
  • Higher Education Institutions

2.2 Relevance

The datasets were selected because they provide essential geographic information required to achieve the project’s objectives:

  • The Student Housing dataset is vital for identifying where there is a centralised amount of accommodations for students.
  • The Higher Education dataset ensures a comprehensive mapping of higher educational institutions, enabling analysis of their spatial relationship to student housing.

2.3 Limitations

Despite their utility, these datasets have some constraints:

  • Data Accuracy: Some entries may be outdated or missing, as datasets are not updated in real time.
  • Coverage: While the datasets aim for nationwide coverage, certain regions have incomplete or inconsistent information.

3.1 Software and Libraries

The following tools and libraries were used for data processing, manipulation, and visualisation in this project:

  • Data Processing:
    • The main focus of data processing was on handling user inputs, addresses, and search terms. JavaScript was used to handle these inputs dynamically, process search queries, and manage the filtering of student housing options based on services such as Wifi, parking, and more.
    • Additionally, data for the map markers (student housing, schools, etc.) was processed using JavaScript arrays and objects, updating the DOM based on user selections.
  • Visualisation:
    • Leaflet.js: This library was used to visualise the map. It allows the integration of OpenStreetMap tiles and other map-related interactions, such as zooming, panning, and handling click events on markers.

3.2 Map Base Layer

OpenStreetMap tiles were used as the base map for this project. OpenStreetMap (OSM) provides free, editable map data, which is perfect for embedding maps in websites or applications. The OSM tiles are integrated into the map via Leaflet.js. This is done by setting up the base layer with OSM’s tile URL and rendering them on the map.

Regarding licensing, OpenStreetMap's data is available under the Open Database License (ODbL), which allows for free usage, but requires attribution. Specifically, any use of OpenStreetMap tiles must include proper attribution to OpenStreetMap contributors. For this project, the attribution is handled by the Leaflet map, which automatically includes the necessary OpenStreetMap attribution on the map.

3.3 APIs Used

Several APIs were integrated into the project to enhance functionality:

  • OpenStreetMap Tile API:
    • The map base layer utilises the OpenStreetMap Tile API, which serves the map tiles that render the geographical data on the map.
  • OpenRouteService API:
    • The OpenRouteService API was integrated to create routes between locations. This allows users to visualise the distances between their selected destinations (e.g., student housing and schools).
    • This API it was also used to compare routes by generation multiple address pairs in parallel (e.g., calculating distances between various housing options and one school).
  • Nominatim API:
    • The Nominatim API was used to convert addresses into geographical coordinates (latitude and longitude). When a user inputs an address, this API takes the input and returns the corresponding geocoordinates, allowing the locations to be plotted on the map. Nominatim is a free API provided by OpenStreetMap, but it has usage policies and limits to prevent excessive load on their servers.

4.1 Overview

The webpage aims to provide both search and visualisation functionalities, offering an efficient way for users to compare locations based on services and distances. Users can interact with the map, filter options, switch languages, and create routes to optimise their search experience.

4.2 User Features

  • Interactive Map with Data Layers (Student Housing, Universities)

    The interactive map was created using Leaflet.js, allowing users to visualise university and student housing locations dynamically. The map incorporates OpenStreetMap tiles as the base layer and additional data layers for student housing and universities. The GeoJSON datasets containing coordinates for universities and housing are integrated into the map as overlays.

    • Zoom and Pan: Users can zoom in and out to explore locations in detail, and the map supports panning, making it easy to navigate across different regions.
    • Markers: Each university and housing location is represented as a marker on the map. These markers can be added and removed dynamically based on the user’s filtering criteria. When a user selects a particular filter (e.g., Wifi, Parking), only the relevant markers are displayed on the map.
    • Custom markers: Users can add custom addresses by typing them into the search bar, which will place a new marker at the specified location on the map. Additionally, users can add locations by clicking directly on the map, creating a new marker at the selected spot.
    • Pop-ups: Each marker has an associated pop-up that provides additional information about the location (e.g., university name, housing services). These pop-ups are triggered when the user clicks on a marker.
  • Filtering Data by Specific Criteria
    • Users can filter the displayed student housing options based on services like Wifi, parking, laundry, and others. The filtering system is implemented using clickable icons that represent each service. When a user selects a filter, the map updates instantly to reflect the filtered results, displaying only the relevant locations.
    • This functionality is achieved by associating each housing marker with specific service data and toggling the visibility of markers based on the selected filters.
  • Picking the Webpage Language
    • The webpage offers users the ability to switch between English and French.
    • The language switcher is implemented using a simple dropdown menu, which triggers a JavaScript function to change the page’s content dynamically. The language-switcher.js script listens for the change in the dropdown and modifies the text content of the page based on the selected language.
    • The implementation method is straightforward and efficient for a bilingual interface, particularly for webpages with minimal written content. It allows users to seamlessly switch between languages without the need to reload the page.
  • Create Routes and Compare Them
    • The routing functionality allows users to create routes between two or more locations (e.g., from a student housing to a university) using the OpenRouteService API.
    • A feature to compare routes for multiple address pairs is implemented, allowing users to see which route is the most efficient when considering multiple destinations from the same location.

4.3 Data Preparation, Integration and Visualisation

The data for universities and student housing is stored in GeoJSON format. This format is ideal for geographic data and is directly compatible with Leaflet.js.

The GeoJSON files are integrated into the map as data layers. Each location (university or housing) is represented as a point on the map, with associated attributes (e.g., name, address, services available) which some are displayed in the pop-ups when users click on the markers.

4.4 Responsiveness

The webpage has been optimised for different devices, including desktops, tablets, and mobile phones.

  • The CSS media queries were used to adjust the layout and font sizes based on the screen size.
  • The map and filter options are fully responsive, with the map automatically resizing when the window is adjusted. On mobile devices, the map is touch-enabled, allowing users to zoom and pan with their fingers.
  • The language selector, search bar, and filter icons are also optimised for mobile use, ensuring that they are easy to access and interact with on smaller screens.

5.4 Challenges and Solutions

During development, we faced several challenges, from technical issues to design choices. This section outlines the key obstacles encountered, the strategies employed to address them, and the tools and collaboration methods that contributed to overcoming these difficulties.

  • Responsiveness of the Search Bar

    One challenge was ensuring the search bar functioned responsively across different devices and screen sizes. Initially, the layout and behavior were inconsistent on smaller screens. This was addressed by adjusting the CSS styles to ensure the search bar remained accessible and user-friendly regardless of the device.

  • Language Switcher Functionality

    The language switcher posed challenges due to the multiple methods available to implement it. Initially, a translation library was considered, but it presented issues, leading to difficulties in implementation. To resolve this, a simpler approach was adopted using JavaScript to toggle between pre-defined language content without reloading the page. This streamlined the functionality and avoided unnecessary complexity.

  • Pop-up Design and Dataset Issues

    The pop-up feature encountered two main obstacles: inconsistent data within the dataset and issues with the image source links. Many images were missing or led to 404 errors, which caused infinite loops in the browser. Rather than excluding images entirely, a default placeholder image was used for entries with missing or broken links. This ensured that the pop-ups remained visually cohesive and functional, while still displaying actual photos when available.

  • Route Calculation and Comparison

    Developing the route calculation and comparison feature involved multiple challenges:

    • Finding a Suitable API: Identifying an API capable of creating and comparing route pairs proved difficult, as many services required payment. After selecting a suitable free API, understanding its documentation and capabilities required significant effort.
    • Implementation: Integrating the API to perform the desired task of route comparisons required trial and error, especially in ensuring the API returned the necessary data for our use case.
    • Design Challenges: Representing the route comparisons visually on the webpage in a clear and intuitive manner required collaboration and iterative adjustments to align with the project’s goals.

To handle these challenges, we ensured team members stayed informed about any new obstacles and worked together to solve them. Regular communication—whether during scheduled project sessions at school, through text messages, or via calls—and continuous testing helped refine solutions, effectively addressing issues without impacting the project's timeline or objectives.

6.1 Contribution and Insights

This project provides a significant contribution by creating a user-friendly, interactive map that visualises the spatial relationship between student housing and universities. The tool offers practical value for students seeking housing, urban planners identifying areas for infrastructure development, and policymakers addressing accessibility and housing availability issues.

Key insights include:

  • The proximity of student housing to universities in urban areas, which facilitates convenience for students.
  • Identified gaps in housing availability near certain campuses, suggesting potential areas for development.
  • The value of routing and filtering features in helping users make informed decisions about housing and transportation options.

6.2 Limitations

Despite its functionality, the project has several limitations:

  • Data Quality and Reliability:

    The accuracy of the data depends on the GeoJSON datasets and the APIs used. Any outdated or incomplete data may affect the map's reliability.

  • Static Maps and Updates:

    The map currently relies on static data layers. It does not integrate dynamic updates, limiting its ability to reflect real-time changes in housing or university data.

  • Public Transport Routing:

    The isn't any availability of public transport options via the OpenRouteService API for the studied region, reducing the effectiveness of the routing feature.

  • Filtering Features:

    While basic filters (Wifi, parking, etc.) are implemented, more complex filtering options (e.g., price range, distance from universities) are not yet available.

6.3 Future Improvements

To enhance the project's functionality and value, several future improvements can be made:

  • Dynamic Data Integration:

    Automating updates to housing and university data would improve accuracy and keep the map current.

  • Export Options: Adding the ability for users to export search results (e.g., a list of housing options) to formats like CSV or PDF would increase the utility of the page.
  • Advanced Filtering: More advanced filtering options could be added, allowing users to filter by additional criteria such as price, distance from universities, or specific amenities. Implementing these features could provide users with more precise results.
  • Public Transport Integration: Incorporating additional APIs for public transportation, would address current limitations and drastically improve this functionality.
  • Additional Data Layers:

    Including other relevant layers, such as grocery stores, libraries, or recreational facilities, could provide a more comprehensive view for users.

  • User Authentication: Adding a user authentication system (e.g., login or registration) could allow users to save their searches and preferences for future use.

6.4 Final Reflections

This project aimed to simplify and enhance the process of finding suitable student housing near universities. Through careful integration of GIS technologies, APIs, and interactive web development techniques, this project fulfils it's main purpose to help users search for housing options and universities, visualise them on an interactive map, filter results by specific criteria, and generate routes between locations.

The results demonstrate the practical benefits of combining geospatial data with user-friendly design, offering value to students, planners, and policymakers alike. While there are limitations, the project serves as a foundation for further developments that could significantly broaden its scope and impact. Ultimately, this tool highlights the potential of geospatial technologies in addressing real-world challenges and improving decision-making processes.
